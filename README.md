简历批量处理工具
自动批量处理PDF格式的简历文件：智能识别文本型/图片型PDF、自动提取文字内容（图片型走OCR）、
自动脱敏手机号与邮箱、自动过滤平台水印、生成汇总文本并自动复制到剪贴板，方便直接粘贴给大模型分析。

目录结构
resumehelper/

├── resume_processor.py      # 【主程序入口，运行这个文件】

├── config.json               # 配置文件（路径、脱敏规则、水印过滤规则等）

├── llm_prompt.txt             # 汇总文件开头会附带的提示词，可自行编辑

├── input_pdfs/                # 【把待处理的PDF简历放这里】

├── output/                    # 处理结果自动生成在这里

   ├── 简历汇总文本_<时间戳>.txt   # 【所有简历合并成一份文本，方便一次性粘贴给大模型】

   └── 简历汇总_<时间戳>/

       ├── pdf/                # 处理成功的PDF原件会被移动到这里备份

       ├── txt/                # 每份简历单独提取出的文本

       └── <时间戳>.log         # 本次运行的详细日志





## 环境准备

### 1. 创建并激活虚拟环境（首次配置时执行）

首次在一台电脑上部署本工具时，建议新建一个独立的Python虚拟环境，避免和系统全局Python环境
互相干扰（尤其是MinerU的依赖较多较重）。在项目根目录下执行：

```powershell
py -3.12 -m venv mineru_env
这一步【只需要执行一次】。如果mineru_env文件夹已经存在（比如你是接手别人已经配置好的环境），
说明虚拟环境已经建好了，跳过这一步即可。只有在环境损坏需要重建、或者换了新电脑时，才需要再执行一次。

【每次要运行程序前，都需要重新执行】——激活虚拟环境：

mineru_env\Scripts\activate
激活成功后，命令行提示符前面会出现 (mineru_env) 标记，例如：

(mineru_env) PS C:\resumehelper>
注意：这个"激活"状态只在【当前这一个命令行窗口】里生效。只要你关闭窗口，或者新开一个
PowerShell窗口，就必须重新执行一次mineru_env\Scripts\activate，否则运行主程序时
用的会是系统全局Python，可能因缺少依赖包而报错。

如果执行激活命令时报错（提示类似"无法加载，因为在此系统上禁止运行脚本"）：

这是Windows PowerShell默认的安全策略限制，不是本程序的问题。解决方法是先执行：

Set-ExecutionPolicy Unrestricted -Scope Process
这条命令只对当前窗口临时放开限制，关闭窗口后失效，不会永久改变系统安全设置，可以放心使用。
执行完这条命令后，再重新执行一次mineru_env\Scripts\activate即可。

2. 安装Python依赖
激活虚拟环境后（确认提示符前有(mineru_env)），执行：

pip install pymupdf pyperclip
pymupdf（即代码里的fitz）：用于判断PDF类型、提取文本型PDF的文字内容
pyperclip：用于把汇总结果自动复制到系统剪贴板（可选，不装也能正常跑完整流程， 只是最后不会自动复制，需要手动打开汇总txt复制）
3. MinerU环境（仅图片型PDF/扫描件需要）
如果你处理的简历里有扫描件、截图生成的PDF（即"图片型PDF"），需要依赖MinerU做OCR文字识别。
这部分环境搭建请参考MinerU官方文档，在mineru_env虚拟环境里单独安装配置好。

如果你确定自己所有的简历都是"文本型PDF"（可以直接在PDF里选中复制文字的那种），
可以不折腾这一步，程序会自动跳过OCR分支。

使用步骤（日常使用，每次都要做的）
第一步：打开命令行，进入项目目录并激活环境
复制
cd C:\resumehelper\
mineru_env\Scripts\activate
第二步：把简历PDF放进输入文件夹
把所有要处理的PDF简历，直接扔进 input_pdfs/ 文件夹（只处理该文件夹最顶层的PDF，
不会去翻子文件夹）。

第三步：（可选）检查配置文件
打开 config.json，一般不需要改，除非你要调整以下内容：

配置项	作用	一般何时需要改
input_folder	输入文件夹路径	想换个存放PDF的位置
output_folder	输出文件夹路径	想换个结果存放位置
char_threshold	判断"文本型/图片型"的字符数阈值	极少需要改，默认50足够
phone_placeholder / email_placeholder	手机号/邮箱脱敏后替换成的文字	想改提示语的措辞
watermark_strings	需要精确删除的水印字符串列表	发现新的、无规律的水印时
watermark_patterns	用正则表达式批量过滤同类水印	发现某类水印有规律时（推荐优先用这个）
llm_api_config.enabled	是否真正调用大模型API（目前是占位，未启用）	未来接入真实API时

第四步：运行程序

python resume_processor.py
程序会自动：

扫描 input_pdfs/ 里所有PDF
逐份判断类型、提取文字、过滤水印、脱敏手机号邮箱
把每份简历单独存成一个txt（在output/简历汇总_xxx/txt/）
把所有简历合并成一份汇总文本（在output/简历汇总文本_<时间戳>.txt）
把处理成功的PDF原件挪到output/简历汇总_xxx/pdf/备份（原始输入文件夹里就不会重复堆积了）
自动把汇总文本内容复制到剪贴板，运行结束后直接找个地方 Ctrl+V 粘贴即可
第五步：查看结果
想看合并后的完整文本（推荐直接拿去粘贴给大模型分析）：打开 output/简历汇总文本_<时间戳>.txt
想看某一份简历单独的提取结果：打开 output/简历汇总_<时间戳>/txt/ 里对应的文件
如果某份PDF处理失败了：控制台和日志会明确提示文件名+失败原因，该PDF会原样留在input_pdfs/里， 不会被误删或误移动，方便你手动检查（常见失败原因：PDF加密、文件损坏）
常见问题
Q: 每次都要重新执行mineru_env\Scripts\activate吗？

是的，只要你关闭了命令行窗口或者新开了一个窗口，就需要重新执行一次。只有"创建"虚拟环境
（py -3.12 -m venv mineru_env）这一步是一次性的，"激活"是每次新窗口都要做的。

Q: 运行后发现汇总文件里还有一段乱码字符串（比如很长一串字母数字加波浪号结尾）？

这通常是某些简历发布平台自动嵌入的追踪水印。可以打开 config.json，把该字符串加入
watermark_strings 列表精确删除；如果发现同类水印反复出现、只是中间一小段随机变化，
建议改用 watermark_patterns 写一条正则表达式统一过滤，一次性覆盖所有同类未来水印，
无需每次手动添加。

Q: 剪贴板没有自动复制内容？

确认是否执行过 pip install pyperclip。如果不装这个库，程序依然会正常生成所有文件，
只是最后一步"自动复制"会跳过，控制台会提示你手动打开文件复制。

Q: 想处理图片型PDF（扫描件），但报MinerU相关错误？

确认mineru_env虚拟环境已经激活（提示符前有(mineru_env)），且该环境里已正确安装配置好
MinerU相关依赖。

免责声明
本工具仅做本地文本处理，不会将简历内容上传到任何第三方服务器
（除非你在llm_api_config中主动启用并配置了真实的大模型API）。
脱敏功能（手机号/邮箱过滤）已经过测试，但建议处理正式数据前，先用少量样本自行核对结果，
确保符合你自己的隐私保护要求。
